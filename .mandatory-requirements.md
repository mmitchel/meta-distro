# Mandatory System Requirements - Meta-Distro Boot Architecture
**Effective Date**: February 16, 2026
**Status**: PERMANENT - ALL FUTURE WORK MUST COMPLY

## 1. GPT Partition Type GUIDs (Discoverable Partitions)

| Partition | GUID | Identifier | Encryption | Mountpoint |
|-----------|------|------------|-----------|-----------|
| EFI System | `c12a7328-f81f-11d2-ba4b-00a0c93ec93b` | SD_GPT_ESP | Never | /boot/efi |
| Boot Loader (XBOOTLDR) | `bc13c2ff-59e6-4262-a352-b275fd6f7172` | SD_GPT_XBOOTLDR | Never | /boot |
| Root x86-64 | `4f68bce3-e8cd-4db1-96e7-fbcaf984b709` | SD_GPT_ROOT_X86_64 | Always LUKS | / (in LVM) |

| Partition/Volume | Filesystem UUID | Purpose |
|------------------|-----------------|---------|

## 3. Partition Discovery Rules

- **Physical Partitions**: Discovered by GPT partition type GUID (not by /dev/sdaX device names)
- **Logical Volumes**: Discovered by filesystem UUID (not by LVM VG/LV names)
- **No Device Name References**: Kernel commands, fstab entries, and initramfs scripts must use UUIDs and GUIDs exclusively

## 4. Initramfs Requirements

The initramfs boot sequence MUST:

1. Locate the LUKS partition by querying GPT partition table for `PARTTYPE=4f68bce3-e8cd-4db1-96e7-fbcaf984b709`
2. Unlock the LUKS container (prompt for passphrase or use TPM-sealed key)
3. Activate LVM physical volume inside LUKS
4. Mount root LV using UUID `8e8c2c0a-4f0e-4b8a-9e1a-2b6d9f3e7c11` to `/sysroot`
5. Execute `/sysroot/usr/lib/ostree/ostree-prepare-root /sysroot`
6. Switch root (`switch_root`) into the OSTree deployment

## 5. Kernel Command Line Requirements

Must include:
```
root=UUID=8e8c2c0a-4f0e-4b8a-9e1a-2b6d9f3e7c11
```

Additional required parameters (for boot orchestration):
- `systemd.unified_cgroup_hierarchy=1` (cgroups v2)
- `cgroup_no_v1=all` (disable cgroups v1)
- `rd.luks.name=<LUKS_UUID>=cryptroot` (for initramfs LUKS unlock)

## 6. WIC Layout Requirements

- Must use `.wks.in` template files with variable substitution
- All GUID placeholders must be substituted with values from this requirements document
- ESP partition must use `c12a7328-f81f-11d2-ba4b-00a0c93ec93b`
- XBOOTLDR partition must use `bc13c2ff-59e6-4262-a352-b275fd6f7172`
- Root partition must use `4f68bce3-e8cd-4db1-96e7-fbcaf984b709`
- Filesystem UUIDs must be set explicitly in WIC plugin or mkfs commands

Example WIC syntax:
```
part /boot/efi --fstype=vfat --partuuid=@ESP_PARTUUID@ \
  --uuid @ESP_UUID@ --part-type c12a7328-f81f-11d2-ba4b-00a0c93ec93b

part /boot --fstype=ext4 --partuuid=@XBOOTLDR_PARTUUID@ \
  --uuid @XBOOTLDR_UUID@ --part-type bc13c2ff-59e6-4262-a352-b275fd6f7172
```

## 7. OSTree Boot Flow

The complete boot sequence must follow:

```
1. EFI Firmware
   ↓
2. systemd-boot (from ESP)
   ↓
3. Kernel + Initramfs (from XBOOTLDR)
   ↓
4. Initramfs /init script
   ├─ Query GPT: Find partition with PARTTYPE=4f68bce3-e8cd-4db1-96e7-fbcaf984b709
   ├─ Unlock LUKS container
   ├─ Activate LVM PV
   ├─ Mount root LV to /sysroot using UUID=8e8c2c0a-4f0e-4b8a-9e1a-2b6d9f3e7c11
   └─ Mount /var to /sysroot/var using UUID=d3b4a1f2-6c9e-4f8b-9c22-0f7b8e1a4d55
   ↓
5. Execute ostree-prepare-root
   ├─ Select active OSTree deployment
   └─ Prepare deployment root for switch_root
   ↓
6. switch_root → /sysroot
   ↓
7. Deployment's /sbin/init
   ├─ systemd startup
   ├─ Mount /var (if not already mounted by initramfs)
   ├─ Standard systemd boot sequence
   └─ System ready
```
